# Decisions

使用此目录保存简洁的 ADR 风格记录，用于记录稳定的技术决策。

## 何时记录决策
- 团队在多个架构或工程方向中做出了选择
- 相关理由应在当前计划关闭后仍可搜索
- 未来贡献者很可能再次回到同一权衡点

## 每条记录保持简短
一份好的决策记录通常包括：
- title and date
- status
- context
- decision
- consequences

## 与其他文档的关系
- 临时性的执行细节放在 `docs/plans/`。
- 稳定的仓库规则放在 `docs/standards/`。
- 当长期架构选择及其理由重要时，把它记录在这里。
