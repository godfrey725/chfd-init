# 测试规则

## 测试策略
- 当行为发生变化时，新增或更新测试。
- 优先编写覆盖真实行为的测试，而不是只验证 mock 的测试。
- 在可行时，让每个用例只聚焦一个行为。
- 在可能的情况下，先用测试复现 bug，再修复它。

## 验证
- 先运行最窄、最相关的测试。
- 当共享行为或接口发生变化时，再运行更广的验证。
- 记录你验证了什么，以及哪些部分无法验证。
- 如果某种失败模式反复重要，就把它沉淀成永久性的回归检查。

## 维护
- 当被覆盖的行为已不存在时，删除过时测试。
- 保持 fixtures 和 helpers 小而易懂。
- 优先使用其他代理也能直接运行、无需隐藏环境准备的测试和命令。
