# Tech Debt Plans

记录那些重要但尚未排期的改进项。

## 此目录用于
- 对当前任务有价值但并非必需的清理工作
- 需要有意识跟进的架构问题
- 反复出现、应转化为未来项目的摩擦点

## 预期
- 说明这项债务为什么重要。
- 描述它可能影响的范围。
- 优先给出具体的后续建议，而不是模糊抱怨。
- 当有助于解释优先级时，链接相关计划、事故或评审反馈。

## 建议条目结构
- title
- why it matters
- current cost or risk
- affected areas
- recommended next step
- any known owner or trigger for revisiting
