# Active Plans

将仍在进行中的工作计划存放在这里。

## 何时创建计划
当工作满足以下情况时，创建一个可持续计划：
- 跨越多个有意义的步骤
- 在多个文件中改变行为
- 需要跨会话或跨协作者交接安全
- 带有架构或运维风险

对于较小但非简单的工作，一条轻量级单消息计划就足够了。只有当计划需要在下一次回复之后仍保持可见时，才使用此目录。

## 预期
- 每个有意义的项目或任务组对应一个计划。
- 使用包含日期和主题的清晰文件名。
- 随着工作推进，及时更新状态。
- 当实现路径、范围或风险变化时，更新已经过期的计划。

## 建议内容
- goal
- constraints
- files involved
- implementation steps
- verification steps
- open risks or follow-ups

## 生命周期
- 在进行大量实现前先创建计划。
- 在执行期间保持复选框或状态标记是最新的。
- 工作完成后，将计划移动到 `../completed/`。
- 不再属于当前活动计划的延期后续项，移动到 `../tech-debt/`。
