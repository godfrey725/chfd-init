# Tech Debt Plans

Capture known improvements that matter but are not yet scheduled.

## Use This Directory For
- cleanup that is valuable but not required for the current task
- architectural issues that need deliberate follow-up
- repeated friction that should become a future project

## Expectations
- Explain why the debt matters.
- Describe the likely area of impact.
- Prefer concrete follow-up ideas over vague complaints.
- Link related plans, incidents, or review feedback when they help explain priority.

## Suggested Entry Shape
- title
- why it matters
- current cost or risk
- affected areas
- recommended next step
- any known owner or trigger for revisiting
