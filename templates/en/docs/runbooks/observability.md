# Observability Runbook

Document how to inspect system health and behavior using logs, metrics, traces, dashboards, and alerts.

## Include
- where logs, metrics, traces, and alerts live
- which signals matter most for normal health
- how to investigate a suspected regression after deploy
- common dashboards or queries for major flows
- known gaps in instrumentation

## Expectations
- Prefer exact links, commands, queries, or dashboard names when available.
- Distinguish between fast triage steps and deeper investigation.
- Update this file when instrumentation or operational signals change.
