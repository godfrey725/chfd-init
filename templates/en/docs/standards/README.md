# Standards

This directory holds the repository's stable operating rules.

## Read These First
1. `golden-rules.md`
2. `coding-rules.md`
3. `testing-rules.md`
4. `logging-rules.md`
5. `review-checklist.md`

## Purpose
Use these docs to reduce repeated prompting and make review expectations explicit.

## Evolution Policy
- Keep standards focused on rules that should survive across many tasks.
- Move task-specific direction into plans instead of standards.
- When repeated review feedback appears, promote it into docs, scripts, lint rules, or tests.
- Remove or rewrite rules that are vague, stale, or no longer enforced in practice.

## Authoring Guidance
- Keep rules brief, concrete, and enforceable.
- Prefer project-wide decisions over task-specific notes.
- Write rules so agents can act on them without guessing hidden context.
- Avoid copying the same rule into every file; link related guidance instead.
