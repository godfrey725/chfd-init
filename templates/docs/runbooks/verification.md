# Verification Runbook

Document how to verify work in this repository before it is considered complete.

## Include
- exact test and validation commands
- when to run narrow versus broad checks
- required manual checks, if any
- how to report unverified areas and remaining risk
- where to find observability signals that confirm the system is healthy

## Expectations
- Prefer repeatable commands.
- Keep verification aligned with actual tooling in the repo.
- Update this file when the expected validation workflow changes.
- Make it obvious which checks are required versus situational.
