# Debugging Runbook

Document the fastest safe ways to investigate problems in this repository.

## Include
- where logs appear
- common failure modes
- useful commands for narrowing issues
- how to reproduce important classes of bugs
- which docs, domains, or runbooks usually help next

## Expectations
- Prefer concrete debugging steps over generic advice.
- Link to architecture or standards docs when they help explain the issue.
- Update this file when the team finds a repeated debugging pattern.
- Turn recurring incidents into durable checks, docs, or follow-up debt.
