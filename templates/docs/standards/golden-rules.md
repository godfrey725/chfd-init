# Golden Rules

## Repository Rules
- Read the relevant docs before making non-trivial changes.
- Keep edits as small as possible while still solving the task.
- Follow existing patterns unless the docs explicitly change them.
- Write a plan before non-trivial implementation.
- Verify changed behavior before declaring success.
- Update documentation when project rules, architecture, or workflows change.

## Decision Rules
- Prefer clarity over cleverness.
- Prefer documented behavior over assumptions.
- Prefer explicit boundaries and responsibilities over implicit coupling.
- Prefer deletion over compatibility layers that no longer serve a need.
- Prefer the smallest worthwhile cleanup over unrelated refactoring.

## Delivery Rules
- Do not report work complete without fresh verification evidence.
- Call out what was verified, what was not verified, and any remaining risks.
- When a task needs durable execution context, record it in `docs/plans/active/`.
