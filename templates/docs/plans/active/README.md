# Active Plans

Store plans here for work that is still in progress.

## When to Create a Plan
Create a durable plan when work:
- spans multiple meaningful steps
- changes behavior across multiple files
- needs handoff safety across sessions or contributors
- carries architectural or operational risk

A lightweight one-message plan is enough for small non-trivial work. Use this directory when the plan should remain visible after the next reply.

## Expectations
- One plan per meaningful initiative or task group.
- Use clear filenames that include date and topic.
- Keep status current as work evolves.
- Update stale plans when the implementation path, scope, or risks change.

## Suggested Contents
- goal
- constraints
- files involved
- implementation steps
- verification steps
- open risks or follow-ups

## Lifecycle
- Create the plan before substantial implementation.
- Keep checkbox or status markers current during execution.
- Move the plan to `../completed/` when the work is done.
- Move deferred follow-up items into `../tech-debt/` when they no longer belong in the active plan.
